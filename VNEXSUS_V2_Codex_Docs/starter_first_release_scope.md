# Starter 첫 릴리스 범위

## 목적

이 문서는 설계사 대상 Starter의 첫 유용한 릴리스가 정확히 어디까지여야 하는지 정의한다.

여기서의 목표는 “모든 기능”이 아니라 “설계사가 실제 케이스를 정리하고 다음 행동을 결정할 수 있는 최소 유용 범위”다.

## 1. Starter v1에서 반드시 있어야 하는 화면 / 결과 블록

### A. 케이스 입력 / 문서 업로드 기본 흐름

필수:

- 케이스 생성
- 문서 업로드
- 피보험자 기본정보 입력
- 보험가입일 입력

이유:

- insuranceJoinDate는 고지의무 검토 overview의 전제다.
- 이미 저장소에는 관련 API와 patient input 구조가 있다.

### B. Starter 분석 개요 화면

필수 블록:

- 사건 기본정보
- 문서 수 / 병원 수 / 기간 범위
- source quality notice

설계사에게 필요한 이유:

- 업로드가 제대로 됐는지
- 어떤 병원과 기간이 걸쳐 있는지
- 결과 신뢰도에 주의가 필요한지

### C. 의료 이벤트 타임라인 블록

필수 표시:

- 날짜
- 병원
- 이벤트 유형
- 짧은 핵심 내용
- review-needed 여부
- 대표 evidence 링크 또는 근거 진입점

Starter v1에서 특히 중요:

- 외래 / 입원 분리
- 수술 / 검사 / 병리 강조
- 한 번에 사건 흐름이 보이는 정렬

### D. 질환군 개요 블록

필수 항목:

- 암
- 심장
- 뇌 / 뇌혈관
- 수술
- 입원
- 만성질환 / 기타 중요 소견

표시 수준:

- 있음
- 없음
- 검토 필요

### E. 고지의무 검토 개요 블록

필수 항목:

- 보험가입일 입력 여부
- 적용 window label
- 후보 이벤트 수
- review-needed 후보 수
- 비판단형 안내 문구

Starter v1 원칙:

- overview까지만
- 확정 판단 금지

### F. 주의 / 경고 블록

반드시 포함:

- 의료적 최종 진단 아님
- 보험금 승인 / 거절 판단 아님
- 문서 품질 및 누락에 따라 결과가 달라질 수 있음
- review-needed 항목은 사람이 다시 확인해야 함

### G. compact summary / 공유 출력

필수:

- 사건 개요
- 핵심 날짜
- 주요 질환군
- 고지의무 검토 개요
- 경고 문구

## 2. Starter v1에서 꼭 들어가야 하는 경고 문구

Starter v1은 다음 취지의 문구를 반드시 고정해야 한다.

- 본 결과는 의료적 최종 진단이 아닙니다.
- 본 결과는 보험금 승인 또는 거절 여부를 판단하지 않습니다.
- 업로드 문서의 품질, 누락, 스캔 상태에 따라 결과가 달라질 수 있습니다.
- 검토 필요 표시가 있는 항목은 사람이 원문을 다시 확인해야 합니다.

## 3. Starter v1에서 없어도 되는 것

아래는 첫 릴리스에 없어도 된다.

- selective vision cross-check
- Pro 질문 / 검색 / 재질문
- 질환군별 깊은 해석
- 고지의무 후보별 detailed reasoning
- Google / Kakao 로그인
- 실제 결제
- entitlement 자동 제어

## 4. Pro로 넘겨야 하는 것

아래는 Starter가 아니라 Pro로 넘겨야 한다.

- 왜 이렇게 분류됐는지 설명형 탐색
- source evidence drill-down 강화
- 모호한 page/block에 대한 selective vision 보강
- 고지의무 후보 상세 검토
- 더 깊은 질환군 해석

## 5. Starter v1에서 필요한 최소 구현 성격

Starter v1은 새 엔진을 만드는 제품이 아니다.

필요한 것은:

- 기존 코어 엔진 결과를 planner-facing 구조로 조립
- 기존 report / narrative / export 기반을 compact summary에 재사용
- review-needed / unresolved 신호를 숨기지 않는 표시 레이어

## 6. v1에서 빠져도 “쓸 수 있는” 이유

Starter v1은 아래 질문에 답할 수 있으면 현장 가치가 있다.

- 사건의 주요 흐름이 무엇인가
- 언제 어느 병원에서 핵심 이벤트가 있었는가
- 암 / 심장 / 뇌 / 수술 / 입원 중 무엇이 보이는가
- 보험가입일 기준으로 검토 후보가 있는가
- 사람이 다시 봐야 할 위험 구간은 어디인가

이 다섯 가지가 되면, 설계사는 고객 응대 전 1차 검토를 할 수 있다.

## 7. 첫 릴리스 해석

Starter 첫 릴리스는 “완전한 상품화 완료”가 아니라 아래 의미다.

- 코어 엔진을 설계사가 실제로 읽을 수 있는 형태로 묶은 첫 usable release
- planner-facing product identity가 처음 드러나는 release
- Pro와 productization으로 가기 전에 가장 먼저 시장 가치를 보여주는 release
