# 인증 / 역할 / 플랜 / 과금 / 관리자 정책서 v1

## 1. 역할 정의
### consumer
일반사용자 — 자기 의료문서 점검, 정밀확인, 전문가연결

### investigator
조사자/실무자 — 보고서 작성, evidence 검토, 업무 생산성 향상

### admin
운영자 — 사용자, 플랜, 사용량, 실패건, 결제, 연결요청 관리

## 2. 회원가입 흐름
### 공통
- 이메일/비밀번호 또는 magic link
- 필수 동의 체크
- 역할 선택

### 일반사용자
- 기본 가입
- 일반 약관/민감정보 동의
- 결과 안내 동의

### 조사자
- 역할 신청
- 실무자 인증 단계
- 승인 전 제한 기능
- 승인 후 Starter/Pro/Studio 부여

## 3. 역할별 UX 분기
### 일반사용자
- 홈: 업로드, 미리확인 CTA
- 결과: 리스크 신호, 추가 확인 포인트
- 업셀: 정밀확인, 전문가연결

### 조사자
- 홈: 케이스 목록
- 케이스: 환자/가입일 입력
- 결과: 이벤트/보고서/evidence 탭
- 내보내기/재분석/사용량 확인

## 4. 플랜 정의
### 일반사용자
#### 미리확인
- OCR 기본 분석
- 페이지 단위 차감
- 추가 분석 미포함
- 업로드 후 “정확도 향상” 업셀 가능

#### 정밀확인
- OCR + 선택적 정밀분석
- 더 높은 비용 또는 패킷 차감
- evidence/날짜 판정 보강

#### 전문가연결
- 연결 요청 생성
- 요약 패키지 생성
- 연결 상태 저장

### 조사자
#### Starter
- 기본 OCR
- 기본 분석
- 제한된 export
- 페이지 기반 사용량

#### Pro
- 월 구독
- 정밀분석 사용
- 조사자 보고서 전체
- evidence click
- export 확대

#### Studio
- 월 구독
- 팀/운영 기능
- 엄격 모드
- 추가 관리자 기능
- 초과 사용량 과금 가능

## 5. 사용량 정책
### 외부 노출 기준
- 처리 페이지 수
- 이번 작업 예상 차감량
- 정밀확인 추가 여부
- 남은 크레딧/패킷 또는 구독 상태

### 내부 ledger 기준
- `ocr_pages_used`
- `precision_pages_used`
- `strict_review_runs`
- `export_count`
- `storage_days`
- `connection_requests_used`

## 6. 업셀 UX
### 일반사용자
“일부 페이지에서 날짜와 진료 의미가 복합적으로 섞여 있습니다.
추가 분석을 진행하면 정확도를 높일 수 있습니다. 진행하시겠습니까?”

### 조사자
“모호성이 높은 이벤트가 감지되었습니다.
정밀 분석을 실행하면 날짜 유형과 근거 연결 정확도를 높일 수 있습니다.”

## 7. 결제 정책 초안
### 일반사용자
- 미리확인: 건별/패킷형
- 정밀확인: 추가 결제 또는 패킷 업셀
- 전문가연결: 별도 연결 수수료 또는 후속 수익모델

### 조사자
- Starter: 사용량 중심
- Pro: 월 구독 + 기본 한도
- Studio: 월 구독 + 팀 기능 + 추가 사용량

## 8. 관리자페이지 기능
### 사용자 관리
- 계정 상태
- 역할
- 플랜
- 승인/정지
- 사용량 조정

### 작업 관리
- OCR 작업 상태
- 실패건
- 재시도
- 사람 검토 큐

### 결제/구독 관리
- 구독 상태
- 결제 이력
- 환불 요청
- 사용량 상세

### 전문가연결 관리
- 연결 요청 목록
- 상태 변경
- 실무자 배정(후순위)

## 9. 권한 정책
- consumer는 자기 케이스만 접근 가능
- investigator는 자기/소속 조직 케이스만 접근 가능
- admin만 운영 콘솔 접근 가능
- export/download 는 플랜별 제한 가능
- 정밀분석은 플랜/사용량 체크 후 실행

## 10. 공개 오픈 전 필수 체크
- 민감정보 동의 최소본 활성화
- 약관/처리방침 공개
- 결제 정책 명시
- 삭제 요청/계정 탈퇴 경로 준비
- 관리자 로그 보존
