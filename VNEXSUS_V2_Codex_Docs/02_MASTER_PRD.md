# VNEXSUS V2 Master PRD

## 1. 제품명
VNEXSUS V2

## 2. 제품 비전
비정형 의료문서를 날짜-이벤트 중심으로 구조화하고, 근거와 함께 검증 가능한 결과를 제공하여
조사자에게는 업무 보고서 자동화 도구를, 일반사용자에게는 자기점검 및 전문가 연결 진입점을 제공한다.

## 3. 문제 정의
### 3.1 조사자/손해사정 실무자
- 의료문서가 PDF/스캔/이미지 형태로 흩어져 있다.
- 날짜가 많고, 내원일/검사일/결과일/입원일/퇴원일이 섞여 있다.
- 병원명/진단명/검사내용/치료내용/의사소견을 수작업으로 정리해야 한다.
- 보고서를 쓰기 위해 다시 원문을 반복 확인해야 한다.
- 근거 추적이 약하면 결과물을 신뢰하기 어렵다.

### 3.2 일반사용자
- 의료문서를 직접 읽기 어렵다.
- 무엇이 문제가 될지 감이 없다.
- 가입일 기준으로 어떤 이력이 눈여겨봐야 하는지 모른다.
- 전문 검토가 필요한지조차 판단하기 어렵다.

## 4. 제품 목표
### 4.1 1차 목표
- 의료문서에서 의미 있는 날짜 중심 이벤트를 정확히 추출한다.
- 모든 핵심 이벤트에 evidence를 연결한다.
- 조사자용 10항목 보고서를 생성한다.
- 일반사용자용 리스크 안내 결과를 생성한다.

### 4.2 2차 목표
- 정밀확인/상위플랜으로 자연스러운 업셀 흐름을 만든다.
- 전문가연결 funnel을 구축한다.
- 관리자페이지/사용량/플랜/과금 체계를 운영 가능 상태로 만든다.

## 5. 제품 범위
### In Scope
- 회원가입/로그인
- 역할별 온보딩
- 파일 업로드
- OCR 처리
- 날짜-이벤트 추출
- evidence 연결
- 조사자용 보고서 출력
- 일반사용자용 리스크 안내
- 플랜/사용량 관리
- 관리자페이지
- 동의/약관 최소 구조

### Out of Scope
- 실시간 전문가 매칭
- 자동 보험금 지급판단 확정
- 완전 자동 판례/약관 해석 엔진
- 대규모 협업 워크플로우
- 모바일 네이티브 앱
- 외부 병원 EMR 직접 연동

## 6. 핵심 사용자
### 조사자 / 실무자
- 손해사정사
- 보험조사 실무자
- 서류 정리 담당자
- 의료문서 보고서 작성 실무자

### 일반사용자
- 자신의 의료문서를 업로드해 위험 신호를 확인하려는 사람
- 고지의무 관련 불안을 사전 점검하려는 사람
- 전문가 상담이 필요한지 먼저 확인하려는 사람

### 관리자
- 사용자/플랜/사용량/결제/실패건을 관리하는 운영자

## 7. 사용자군별 가치 제안
### 조사자
- 날짜별 의료이벤트 자동 정리
- 10항목 경과보고서 생성
- evidence 클릭으로 원문 검증
- 반복 업무 시간 단축
- 모호 케이스 정밀분석 지원

### 일반사용자
- 의료문서 핵심 이력 요약
- 검토 필요 신호 안내
- 가입일 기준 주의 구간 안내
- 정밀확인 업셀
- 전문가연결 신청

## 8. 플랜 구조
### 일반사용자
#### 미리확인
“의료문서를 올리고, 문제가 될 수 있는 진료 이력을 먼저 확인해보세요.”
- OCR 기반 기본 분석
- 주요 날짜/병원/이벤트 정리
- 검토 필요 신호
- 기본 evidence 링크

#### 정밀확인
“모호한 날짜와 진료 의미를 더 정밀하게 분석해 정확도를 높입니다.”
- OCR + 선택적 정밀분석
- 입원/수술/조직검사/치료 분리 강화
- 가입일 기준 주의 구간
- 더 정확한 이벤트 번들링
- 추가 evidence 보강

#### 전문가연결
“정리된 결과를 바탕으로 전문가 상담까지 이어갈 수 있습니다.”
- 전문가 검토 요청서 생성
- 요약 패키지 전달
- 상담 연결 신청

### 조사자
#### Starter
- 기본 OCR
- 기본 이벤트 정리
- 제한적 보고서 출력
- 페이지 기반 과금

#### Pro
- 월 구독
- 정밀 분석
- 10항목 보고서
- click-to-evidence
- 내보내기 확대

#### Studio
- 월 구독
- 팀/관리자 기능
- 고급 검토
- 엄격 모드
- 운영 대시보드

7. 필요 시 정밀분석 실행
8. 10항목 보고서 확인

10. 저장/내보내기

### 일반사용자 여정
1. 회원가입
2. 동의 절차

4. OCR 기반 미리확인 결과 확인
5. “추가 분석으로 정확도를 높일 수 있습니다” 업셀
6. 정밀확인 결과 확인
7. 필요 시 전문가연결 신청

- 사람 검토 큐

- 검토 필요 신호
- 추가 확인 포인트
- 추가 분석 CTA
- 전문가연결 CTA

## 12. 품질 지표(KPI)
### 엔진 품질
- date_precision >= 0.92
- date_type_accuracy >= 0.88
- event_bundle_accuracy >= 0.85
- evidence_attachment_rate >= 0.95
- report_slot_fill_rate >= 0.90

### 제품 품질
- OCR 성공률
- 재분석 전환율
- 보고서 검수 시간 절감률
- 일반사용자 → 정밀확인 전환율
- 일반사용자 → 전문가연결 전환율

## 13. 비기능 요구사항
- 민감정보 암호화 저장
- 업로드/처리/조회 권한 분리
- 감사 로그
- 작업 재시도 가능
- 대용량 문서 처리 가능
- evidence 인덱스 고정성

## 14. 핵심 성공 기준
- 조사자가 원문을 뒤지는 시간을 줄인다.
- 일반사용자가 “무엇을 추가로 확인해야 하는지” 알게 된다.
- 결과가 evidence와 함께 검증 가능하다.
- 플랜 구조가 엔진 품질을 해치지 않는다.
