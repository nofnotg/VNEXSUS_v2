# VNEXSUS V2 Master PRD

## 1. 제품명

VNEXSUS V2

## 2. 제품 비전

VNEXSUS V2는 설계사가 고객의 보험금청구 건을 검토할 때, 여러 의료문서를 빠르게 읽고 핵심 의료 이벤트와 고지의무 관련 후보 진료를 구조적으로 파악할 수 있도록 돕는 의료문서 분석 앱이다.

핵심은 다음 세 가지다.

- 날짜-이벤트 중심으로 의료기록을 구조화한다.
- 모든 중요한 결과를 evidence와 함께 제시한다.
- Starter / Pro 단계별로 분석 깊이를 나누되, 의료적/보험적 최종 판단은 앱이 내리지 않는다.

## 3. 제품 타깃

### 3.1 주 타깃

- 설계사
- 보험금청구 건을 사전 검토해야 하는 현장 영업 조직
- 고객 문서를 빠르게 정리하고 검토 포인트를 파악해야 하는 사용자

### 3.2 보조 타깃

- 내부 검토자
- 운영 관리자

이 제품은 더 이상 “일반사용자용 앱”과 “손해사정조사용 앱”의 이원 구조를 기본 전제로 하지 않는다.

## 4. 해결하려는 문제

- 의료문서가 PDF / 스캔 / 이미지로 흩어져 있다.
- 날짜가 많고, 내원일 / 검사일 / 결과일 / 입원일 / 퇴원일 / 보고일이 섞여 있다.
- 병원명 / 진단명 / 검사 / 수술 / 입원 / 병리 / 소견을 수작업으로 정리해야 한다.
- 보험가입일 기준으로 무엇이 고지의무 검토 후보인지 빠르게 보기 어렵다.
- 문서가 많을수록 핵심 사건의 흐름과 근거 추적이 어려워진다.

## 5. 제품 목표

### 5.1 1차 목표

- 의미 있는 날짜 중심 의료 이벤트를 정확히 구조화한다.
- 모든 핵심 이벤트에 evidence를 연결한다.
- 설계사가 바로 읽을 수 있는 Starter 분석 결과를 제공한다.
- 필요 시 Pro 심층 분석으로 확장할 수 있는 구조를 제공한다.

### 5.2 2차 목표

- selective vision cross-check를 통한 상위 검증층을 제공한다.
- 질문/검색/재질문 기능을 통해 Pro 사용성을 높인다.
- 간편 회원가입, 소셜 로그인, 결제/구독 기반 운영이 가능한 제품 구조를 준비한다.

## 6. 핵심 상품 구조

### 6.1 Starter

Starter는 설계사가 현장에서 바로 보는 1차 분석층이다.

제공 가치:

- 전체 사건 개요
- 의료 이벤트 시계열
- 주요 질환군 개괄 정리
- 수술 / 입원 / 검사 / 병리 핵심 요약
- 보험가입일 기준 고지의무 검토 개요
- 누락 가능성 / 편차 / 검토 필요 신호 표시
- 의료적/보험적 판단 금지 안내

### 6.2 Pro

Pro는 Starter 결과를 더 깊게 검증하고 설명하는 심층 분석층이다.

제공 가치:

- Starter 전체 포함
- selective vision cross-check
- 더 깊은 질환군별 검사 해석
- 질문 / 검색 / 재질문 기능
- evidence drill-down 강화
- 고지의무 후보 진료 상세 검토

## 7. 핵심 출력 구조

### 공통 코어 분석 결과

- 문서 통합 현황
- 병원별 / 기간별 문서 수
- 전체 의료 이벤트 타임라인
- 주요 질환군 분류
- 보험가입일 기준 고지 구간 분류
- confidence / review-needed 상태
- evidence 링크

### Starter 출력

- 사건 개요
- 핵심 이벤트 시계열
- 질환군별 개괄 요약
- 고지의무 검토 개요
- 주의사항 및 판단 금지 안내
- 기본 공유용 리포트

### Pro 출력

- Starter 전체
- selective vision 보강 결과
- anchor-level evidence drill-down
- 질의응답 / 검색 / 재질문
- 더 상세한 검토 리포트

## 8. In Scope

- 파일 업로드
- OCR 처리
- 날짜-이벤트 추출
- evidence 연결
- Starter / Pro 구조화 출력
- 설계사용 질문/검색 기반 심층 분석 준비
- 회원가입 / 로그인 / 소셜 로그인 skeleton
- 플랜 / 과금 / 구독 설계
- 관리자 기본 운영 구조

## 9. Out Of Scope

- 보험금 지급 / 부지급 자동 확정
- 의료적 최종 진단
- 보험사 / 의사 판단 대체
- broad consumer app 별도 구축
- 대규모 협업 워크플로우
- EMR 직접 연동
- productization 세부 구현의 조기 착수

## 10. 핵심 사용자 흐름

### Starter 흐름

1. 회원가입 또는 로그인
2. 케이스 생성
3. 피보험자 정보 및 보험가입일 입력
4. 의료문서 업로드
5. OCR 처리
6. 날짜-이벤트 구조화
7. Starter 분석 결과 확인
8. 검토 필요 신호와 고지의무 후보 확인
9. 필요 시 Pro로 확장

### Pro 흐름

1. Starter 결과 기반으로 심층 분석 진입
2. selective vision cross-check 실행
3. 질환군별 / 근거별 drill-down 확인
4. 질문 / 검색 / 재질문 수행
5. 상세 분석 리포트 확인

## 11. 운영 원칙

- 보험가입일은 OCR로 추출하지 않는다.
- evidence 없는 핵심 이벤트는 확정하지 않는다.
- 구조화 JSON을 먼저 만들고, narrative/report는 나중에 만든다.
- 코어 엔진이 productization 기능보다 항상 우선한다.
- 앱은 판단을 대신하지 않고, 검토를 돕는다.

## 12. 성공 기준

### 엔진 기준

- 날짜 추출 품질 안정화
- evidence 연결률 유지
- weak evidence 섹션의 honest signaling 유지
- 핵심 tail-risk 케이스 비회귀

### 제품 기준

- 설계사가 사건 개요를 빠르게 이해할 수 있다.
- Starter 결과만으로도 고객 응대 전 1차 검토가 가능하다.
- Pro에서는 더 깊은 근거 검증과 질문형 탐색이 가능하다.
- 로그인 / 결제 / 구독 구조가 나중에 붙을 수 있도록 문서가 정렬되어 있다.
