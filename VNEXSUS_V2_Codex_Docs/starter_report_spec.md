# Starter 결과 스펙

## 목적

Starter는 설계사가 현장에서 고객 보험금청구 건을 1차로 정리하고 검토 포인트를 빠르게 파악하기 위한 첫 분석 결과다.

Starter는 “가볍게 보이는 화면”이 아니라, evidence 기반 공통 분석 결과를 설계사 실무용으로 압축해 보여주는 층이다.

## Starter가 필요한 상황

- 고객이 의료문서를 여러 장 보냈지만 핵심 흐름이 정리되지 않았을 때
- 설계사가 사건의 큰 줄기와 검토 포인트를 빠르게 파악해야 할 때
- Pro로 넘어갈지 판단하기 전에 1차 신뢰 가능한 분석이 필요할 때

## Starter가 보여주는 결과 섹션

### 1. 사건 기본정보
- 케이스 식별 정보
- 보험가입일 입력 여부
- 문서 수 / 병원 수 / 분석 시점

### 2. 문서 개요
- 병원별 문서 분포
- 기간 범위
- source quality notice

### 3. 전체 의료 이벤트 시계열
- 날짜
- 병원
- 외래 / 입원 / 수술 / 검사 / 병리 / 응급 / 추적관찰 구분
- 짧은 핵심 내용
- 대표 evidence 링크

### 4. 질환군 개요
- 암
- 심장
- 뇌 / 뇌혈관
- 수술
- 입원
- 만성질환 / 기타 중요 소견

표시 방식:

- 있음
- 없음
- 검토 필요

### 5. 고지의무 검토 개요
- 보험가입일 입력 여부
- 적용 가능한 검토 window 개요
- 후보 진료 개수
- 검토 필요 후보 개수
- 깊은 판단이 아니라 overview 성격임을 명시

### 6. 주의 / 변동 가능성 안내
- source quality에 따른 편차 가능성
- 누락 가능성
- review-needed 항목 개수
- unresolved 항목 존재 여부

### 7. 기본 공유 / 출력 결과
- 사건 개요 요약
- 핵심 이벤트 요약
- 질환군 개괄
- 고지의무 검토 개요
- 주의 문구 포함 요약본

## Starter가 요약하는 것

- 전체 사건의 의료 흐름
- 핵심 이벤트와 날짜
- 질환군별 존재 여부와 개괄 수준 설명
- 보험가입일 기준 검토 필요 후보의 overview
- 설계사가 다음 행동을 정하는 데 필요한 최소한의 근거 구조

## Starter가 의도적으로 하지 않는 것

- 의료적 확정 진단
- 보험금 승인 / 거절 판단
- 복잡한 문서 전체에 대한 고비용 심층 cross-check
- 무제한 질문 / 검색 / 재질문
- 전문가 수준의 상세 리포트 드릴다운

## confidence / variance / missing-risk 표시 방식

Starter는 clean summary만 보여주지 않는다. 아래 신호를 함께 보여줘야 한다.

- confidence: high / medium / low
- review-needed 배지
- unresolved 배지
- source quality warning
- 누락 가능성 / 스캔 품질 편차 안내

표시 원칙:

- confidence가 낮은데도 단정형 문장만 보여주면 안 된다.
- review-needed 항목은 “정리됨”으로 위장하지 않는다.
- missing-risk는 분석의 한계로 분리해서 보여준다.

## disclosure-review overview 표시 방식

Starter의 고지의무 검토는 overview다.

보여줄 것:

- 보험가입일 존재 여부
- 후보 이벤트 수
- 기간대별 간단 분류
- 검토 필요 여부
- 후보가 된 이유의 짧은 설명

보여주지 않을 것:

- 최종 위반/비위반 확정
- 법적/보험적 결론 문장

## disease-group overview 표시 방식

Starter의 질환군 출력은 “빠른 상황 이해”가 목적이다.

예시 표현 원칙:

- 암 관련 병리 / 수술 / 입원 소견 있음
- 심장 관련 검사 및 시술 기록 검토 필요
- 뇌혈관 관련 주요 이벤트 미확인

세부 검사 해석은 Pro 영역으로 넘길 수 있다.

## Starter share / export output

Starter는 다음 종류의 파생 출력을 생성할 수 있다.

- 기본 사건 개요 요약
- 내부 검토용 간단 리포트
- 고객 응대 전 내부 공유용 compact summary

모든 출력에는 아래 경고를 포함해야 한다.

- 의료적 최종 진단 아님
- 보험금 승인/거절 판단 아님
- 원문 품질과 문서 누락 여부가 결과에 영향을 줄 수 있음

## 의무 경고 문구

Starter에는 최소한 아래 취지의 문구가 포함되어야 한다.

- 본 결과는 의료 진단서가 아니며 의료적 최종 판단을 대신하지 않습니다.
- 본 결과는 보험금 승인 또는 거절 여부를 판단하지 않습니다.
- 업로드 문서의 품질, 누락, 스캔 상태에 따라 결과가 달라질 수 있습니다.
